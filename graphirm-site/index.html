Why graph-native?

⬡

Cross-session memory

Knowledge nodes persist across sessions. PageRank surfaces what matters — not just what was said last.

↗

Context by relevance

Graph traversal + edge weights build the context window. Relevant history from 10 sessions ago beats verbatim recency.

⊙

Human-in-the-loop

Intercept any tool call before it runs — approve, reject, or rewrite the argument. The decision and its outcome are edges in the graph. No other agent can do this without rebuilding its data model.

◉

Navigate your reasoning

Chat and graph side-by-side in VS Code. Click any node to see what triggered it and what it produced.

⚡

Local knowledge extraction

GLiNER2 ONNX runs on-device in ~600ms per turn. Knowledge nodes are created without an extra API call — fast, private, and offline-capable.

▣

Single Rust binary

No Docker, no Python, no runtime deps. One static binary + SQLite. Runs on any server or laptop.

Architecture

graphirm/ ├── crates/ │ ├── graph/ ← GraphStore (rusqlite + petgraph) — every node + edge │ ├── llm/ ← multi-provider via rig-core (Anthropic, OpenAI, DeepSeek, Ollama…) │ ├── tools/ ← bash, read, write, edit, grep, find, ls │ ├── agent/ ← async loop, context engine, multi-agent, knowledge extraction │ ├── server/ ← axum REST + SSE streaming API │ └── tui/ ← ratatui terminal UI └── graphirm-vscode/ ← VS Code/Cursor extension (chat + d3 graph explorer)

Get started

1

Clone and build (Rust 1.75+ required)

git clone https://github.com/graphirm/graphirm && cd graphirm && cargo build --release
2

Set your LLM API key and start the server

export ANTHROPIC_API_KEY=sk-ant-... && ./target/release/graphirm serve

Supports Anthropic, OpenAI, DeepSeek, Google, and 13 more providers via GRAPHIRM_MODEL.

3

Install the VS Code extension

Download the latest .vsix from Releases, then in VS Code or Cursor: Extensions → ⋯ → Install from VSIX

4

Open the panel

Ctrl+Shift+P → Graphirm: Open
